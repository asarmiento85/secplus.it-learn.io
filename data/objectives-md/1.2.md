# 1.2 — Fundamental Security Concepts

## What you need to know

This objective covers the foundational principles that underpin all of information security. You need to understand the CIA Triad, non-repudiation, the AAA framework, zero trust architecture, physical security controls, and deception/disruption technologies. These concepts appear in scenario questions throughout the entire exam — not just Domain 1. Master these first because everything else builds on them.

---

## The CIA Triad

The **CIA Triad** is the cornerstone model for information security. Every security control exists to protect one or more of these three properties.

| Property | Definition | If Violated... | Controls that Protect It |
|---|---|---|---|
| **Confidentiality** | Only authorized parties can access information | Data breach, unauthorized disclosure | Encryption, access controls, MFA, data classification |
| **Integrity** | Data is accurate, complete, and has not been tampered with | Data corruption, unauthorized modification | Hashing, digital signatures, FIM, version control |
| **Availability** | Systems and data are accessible when needed by authorized users | Outage, DoS attack, ransomware | Redundancy, backups, clustering, load balancing, UPS |

### Confidentiality Controls
- **Encryption** — transforms data so only authorized parties can read it (AES, RSA, TLS)
- **Access Control** — permissions, RBAC, least privilege
- **Data Classification** — labeling data by sensitivity (Public, Internal, Confidential, Restricted)
- **MFA** — prevents unauthorized authentication

### Integrity Controls
- **Hashing** — one-way function producing a fixed digest; if the hash matches, data is unchanged (SHA-256, SHA-3)
- **Digital Signatures** — hash encrypted with sender's private key; proves data hasn't changed and verifies the sender
- **File Integrity Monitoring (FIM)** — tools like Tripwire that alert when files change unexpectedly
- **Version Control** — tracks changes to code/configs; enables rollback

### Availability Controls
- **Redundancy** — duplicate components (power supplies, network paths)
- **Backups** — regular copies of data, tested restoration
- **Clustering** — multiple servers act as one; failover if one fails
- **Load Balancing** — distributes traffic across servers; prevents any one from being overwhelmed
- **UPS (Uninterruptible Power Supply)** — keeps systems running during power outages
- **Disaster Recovery / Business Continuity Plans** — documented recovery procedures

## Non-Repudiation

**Non-repudiation** means a party cannot deny having performed an action. It proves that a message or transaction originated from a specific source.

- **Digital Signatures** — if you signed it with your private key, you cannot claim you didn't send it
- **Audit Logs** — time-stamped, tamper-evident records of who did what and when
- **PKI (Public Key Infrastructure)** — the certificate/key system that makes digital signatures trustworthy
- **Video Surveillance** — non-repudiation in the physical world (you were on camera)
- **Timestamping** — proves when an action occurred

**Key distinction:** Non-repudiation is a *property*, not a single control. It is achieved through a combination of authentication + integrity + logging.

## AAA Framework

AAA stands for **Authentication, Authorization, and Accounting**. It is the framework for controlling access to resources.

| Component | Question it Answers | Examples |
|---|---|---|
| **Authentication** | Who are you? (Prove your identity) | Passwords, biometrics, smart cards, MFA, certificates |
| **Authorization** | What are you allowed to do? | Permissions, ACLs, RBAC, ABAC |
| **Accounting** | What did you do? | Logs, SIEM, audit trails, RADIUS/TACACS+ accounting |

### Authentication Factors
| Factor | Type | Examples |
|---|---|---|
| Something you **know** | Knowledge | Password, PIN, security question |
| Something you **have** | Possession | Smart card, hardware token, OTP app, phone |
| Something you **are** | Inherence / Biometric | Fingerprint, retina scan, facial recognition, voice |
| Somewhere you **are** | Location | IP address, GPS location, geofencing |

**MFA** = two or more different factor *types* (not just two passwords — that is still single-factor).

### Authorization Models
- **RBAC (Role-Based Access Control)** — permissions assigned to roles, users get roles (most common in enterprises)
- **ABAC (Attribute-Based Access Control)** — permissions based on attributes (user department, time of day, resource classification)
- **MAC (Mandatory Access Control)** — labels assigned to subjects and objects; used in government/military (Top Secret, Secret)
- **DAC (Discretionary Access Control)** — resource owner controls permissions; most common in consumer OS (Windows file permissions)

### Accounting
- Logs everything: logins, file access, commands executed, network connections
- **SIEM** (Security Information and Event Management) — aggregates and correlates logs from multiple sources
- **RADIUS / TACACS+** — protocols that provide AAA services for network access

## Gap Analysis

**Gap analysis** compares your organization's *current* security posture to the *desired* or *required* state.

- Identifies: what controls are missing, what controls are insufficient
- Output: a prioritized list of remediation tasks
- Common frameworks used as the "desired state": NIST CSF, ISO 27001, CIS Controls, HIPAA, PCI-DSS
- Process: Assess current state → Define target state → Identify gaps → Create remediation roadmap

## Zero Trust Architecture

**Zero trust** is a security model based on the principle: **"Never trust, always verify."** There is no implicit trust based on network location — even inside the corporate network.

### Core Principles
| Principle | Description |
|---|---|
| **Verify explicitly** | Always authenticate and authorize based on all available data points |
| **Use least privilege** | Limit access to only what is needed (JIT, JEA — Just-In-Time, Just-Enough-Access) |
| **Assume breach** | Design as if attackers are already inside; minimize blast radius |

### Key Components
- **Microsegmentation** — divide the network into small zones; limit lateral movement between them
- **Continuous verification** — identity and device health checked continuously, not just at login
- **Identity-centric security** — identity is the new perimeter; strong authentication for every access attempt
- **Software-Defined Perimeter (SDP)** — access policies enforced dynamically based on identity and context
- **Multi-factor authentication** — core requirement of zero trust

**Traditional (castle-and-moat) model:** Once inside the perimeter, you are trusted.
**Zero trust model:** Location means nothing; every request is verified regardless of source.

## Physical Security Controls

Physical security protects assets from real-world threats — theft, vandalism, unauthorized physical access.

| Control | Purpose | Notes |
|---|---|---|
| **Bollards** | Prevent vehicle ramming attacks against buildings | Concrete/steel posts; prevents cars from driving into a lobby |
| **Access Control Vestibule / Mantrap** | Ensure only one person passes through at a time | Two doors with an airlock; prevents tailgating; often uses badge readers |
| **Fencing** | Defines perimeter; deters and delays unauthorized access | Height matters: 3-4 ft deters casual entry; 6-7 ft deters determined attempts; 8 ft+ with barbed wire is serious |
| **Video Surveillance (CCTV)** | Monitors and records physical activity | Detective and deterrent; PTZ (Pan-Tilt-Zoom) cameras provide coverage |
| **Security Guards** | Human observation and response | Deterrent, detective, and can be preventive; can verify identity |
| **Badge Readers** | Electronic access control for doors | Can be combined with PIN (something you have + something you know) |
| **Lighting** | Deters criminal activity; supports camera effectiveness | Deters threats in low-light areas |
| **Locks** | Prevent unauthorized physical access | Traditional key, electronic keypad, smart lock |

### Tailgating / Piggybacking
- **Tailgating:** Following an authorized person through a secure door without using your own credentials
- **Countermeasure:** Access control vestibule (mantrap), security guards, employee training

## Deception and Disruption Technologies

These technologies detect attackers and gather intelligence by luring them with fake targets.

| Technology | Description | Purpose |
|---|---|---|
| **Honeypot** | A fake, intentionally vulnerable system designed to attract attackers | Detect intrusion attempts; gather attacker intelligence |
| **Honeynet** | A network of honeypots; simulates an entire fake network | Detect and study more sophisticated attacks |
| **Honeyfile** | A fake file (often named something enticing like "passwords.txt") | Triggers an alert if accessed; reveals unauthorized access |
| **Honeytoken** | Fake credentials, API keys, or data embedded in systems | Triggers alert when used; reveals credential theft |

### Key Points
- Honeypots are **legal** to operate — but legal considerations apply to what you do with connecting attackers
- They are **detective** controls — they do not prevent attacks but reveal them
- They waste attackers' time and provide intelligence on TTPs (Tactics, Techniques, and Procedures)
- **Breadcrumbs** — fake data planted to lead attackers to honeypots

## Exam Tips

- **CIA Triad questions are everywhere** — when you see a scenario describing a breach, identify which property was violated first.
- **Availability** is violated by DoS attacks, ransomware, hardware failure, and power outages — not just hackers stealing data.
- **Non-repudiation** is primarily achieved through **digital signatures** — know this cold.
- **AAA** — Authentication proves identity, Authorization grants access, Accounting tracks actions. These are separate steps and often separate systems (RADIUS, TACACS+).
- **MFA must use two different factor types** — two passwords is not MFA; a password + fingerprint is.
- **Zero trust** key phrase: "never trust, always verify" — if you see this phrase, the answer relates to zero trust.
- **Microsegmentation** is the zero trust network architecture technique that limits lateral movement.
- **Mantrap / access control vestibule** = the physical double-door airlock that prevents tailgating.
- **Bollards** specifically protect against vehicle-based physical attacks — not general perimeter control.
- **Honeypot** = single fake system; **Honeynet** = fake network; **Honeytoken** = fake data/credential.
- **Gap analysis** = comparing current to desired security posture — expect questions about compliance frameworks.
- **Least privilege** is a principle of both zero trust and general access control — users get only what they need.
