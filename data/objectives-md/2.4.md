# 2.4 — Indicators of Malicious Activity

## What you need to know

Indicators of malicious activity are observable signs that suggest a system or network has been compromised, is under attack, or is behaving anomalously. The Security+ exam tests your ability to recognize specific indicators and map them to the correct attack or malware type. You need to know both technical indicators (file changes, network traffic patterns, process behavior) and contextual indicators (impossible travel, off-hours logins, missing logs) that appear in SOC analyst scenarios.

---

## Malware Types and Their Indicators

### Ransomware

| Attribute | Detail |
|---|---|
| **What it does** | Encrypts victim's files and demands payment for decryption key |
| **Delivery** | Phishing email, RDP exploitation, drive-by download |
| **Indicators** | Files renamed with unknown extension (.locky, .ryuk), ransom note left in directories, sudden high CPU/disk I/O, inability to open files, shadow copies deleted |
| **Notable examples** | WannaCry, REvil, LockBit, Ryuk, Conti |

- Ransomware-as-a-Service (RaaS) allows low-skill criminals to deploy sophisticated ransomware
- Immediate response: isolate the infected machine to prevent lateral movement

---

### Trojan

| Attribute | Detail |
|---|---|
| **What it does** | Disguises itself as legitimate software; performs malicious actions once executed |
| **Indicators** | Unexpected outbound network connections, new processes not matching installed software, modifications to startup entries |
| **Subtypes** | Remote Access Trojan (RAT), banker Trojan, dropper Trojan |

- RATs give attackers interactive remote control of the victim's system
- Trojans require **user interaction** to execute (unlike worms)

---

### Worm

| Attribute | Detail |
|---|---|
| **What it does** | Self-replicates and spreads across a network without user interaction |
| **Indicators** | Unusually high network traffic, multiple hosts exhibiting same symptoms simultaneously, rapid spread |
| **Notable examples** | WannaCry (worm + ransomware), Slammer, CodeRed |

- Worms exploit network vulnerabilities to spread autonomously — no user action needed
- High network traffic is a classic worm indicator

---

### Spyware

| Attribute | Detail |
|---|---|
| **What it does** | Secretly collects information about the user (browsing habits, credentials, personal data) |
| **Indicators** | Browser redirects, homepage changes, sluggish performance, unauthorized data exfiltration in network logs, pop-up ads |

- Often bundled with free software (adware component)
- Keylogger functionality frequently included

---

### Bloatware

- **Unwanted pre-installed software** on devices (PUP — Potentially Unwanted Program)
- Not malicious by design but consumes resources and may create privacy risks
- Can be a vector for more serious malware
- Indicators: unexpected applications at startup, high resource usage, unknown processes

---

### Virus

| Attribute | Detail |
|---|---|
| **What it does** | Attaches to legitimate files; spreads when infected file is executed/shared |
| **Key trait** | Requires **user action** to spread (unlike a worm) |
| **Indicators** | Unexpected file size changes, file corruption, performance degradation, AV alerts |
| **Subtypes** | Boot sector virus, macro virus, polymorphic virus (changes signature), metamorphic virus |

- Polymorphic viruses mutate to evade signature-based detection
- Fileless viruses operate entirely in memory, leaving no files on disk

---

### Keylogger

| Attribute | Detail |
|---|---|
| **What it does** | Records keystrokes to capture passwords, PINs, credit card numbers |
| **Types** | Software (runs as process), hardware (physical device plugged between keyboard and computer) |
| **Indicators** | Credential theft after no other apparent attack, unknown processes, suspicious registry entries |

- Hardware keyloggers are invisible to software-based AV
- Used in targeted attacks against executives, financial staff

---

### Logic Bomb

| Attribute | Detail |
|---|---|
| **What it does** | Malicious code that executes when a specific condition is met (date, login, event) |
| **Trigger examples** | Specific date/time, user account deletion, file access, login count |
| **Indicators** | Sudden destructive behavior on specific date; difficult to detect before triggering |
| **Insider threat connection** | Often planted by a disgruntled employee as a time-delayed revenge mechanism |

- The bomb is dormant until the trigger condition is met — may exist in code for months before detonation
- Defense: code reviews, separation of duties, change management

---

### Rootkit

| Attribute | Detail |
|---|---|
| **What it does** | Hides itself and other malware at the OS or firmware level; modifies kernel to avoid detection |
| **Types** | User-mode rootkit, kernel-mode rootkit, bootkit (infects boot sector/UEFI), hypervisor rootkit |
| **Indicators** | Discrepancy between running processes as seen by OS vs. external tools; hidden files; AV that can't find what behavioral tools see |
| **Detection** | Offline scanning, anti-rootkit tools (e.g., Rootkit Revealer), integrity checking (Tripwire) |
| **Removal** | Often requires full OS reinstall; bootkits may require firmware reflash |

- Rootkits are the **hardest malware to detect** because they subvert the very tools used to find them
- A kernel-mode rootkit runs at the highest privilege level on the system

## Physical Attacks

| Attack | Description |
|---|---|
| **Brute force (physical)** | Forcing entry — breaking locks, smashing windows, bypassing physical access controls |
| **RFID cloning** | Copying the data from an RFID access card to create a duplicate; bypasses card-based access control |
| **Environmental attacks** | Sabotaging HVAC, power, or cooling systems to damage hardware or cause downtime |

- Physical security is foundational — if an attacker can physically touch a system, many logical controls fail

---

## Network Attacks

### DDoS (Distributed Denial of Service)

| Type | How it Works |
|---|---|
| **Volumetric** | Flood bandwidth with traffic (UDP flood, ICMP flood, DNS amplification) |
| **Protocol** | Exploit protocol weaknesses (SYN flood — exhausts TCP connections) |
| **Application layer (Layer 7)** | HTTP GET/POST floods that exhaust web server resources |

- **DDoS** uses multiple compromised hosts (botnet); **DoS** uses a single source
- Indicators: service unavailability, extremely high inbound traffic, exhausted server connections

### DNS Poisoning / Spoofing

- Corrupting a DNS cache with false records; users are redirected to attacker-controlled IP
- Indicator: users redirected to unexpected sites, SSL certificate mismatches

### On-Path Attack (MITM — Man-in-the-Middle)

- Attacker positions themselves between two communicating parties; intercepts and potentially modifies traffic
- Previously called "Man-in-the-Middle" — CompTIA now uses "on-path attack"
- Indicators: unexpected certificate warnings, unusual ARP table entries (ARP poisoning), slow connections

### Credential Replay

- Attacker captures valid authentication tokens or hashes and replays them to gain access
- **Pass-the-Hash (PtH)**: capture NTLM hash and replay it without cracking the password
- Defense: use Kerberos, implement MFA, use session tokens with short expiry

### Wireless Attacks

| Attack | Description |
|---|---|
| **Evil twin / Rogue AP** | Fake AP with same SSID as legitimate network |
| **Deauthentication attack** | Flood 802.11 deauth frames to disconnect clients (then capture handshake) |
| **WPA2 handshake capture** | Capture 4-way handshake for offline brute force cracking |

---

## Application Attacks

| Attack | Description |
|---|---|
| **Injection (SQLi, XSS, command)** | Malicious code injected into input fields |
| **Buffer overflow** | Writing beyond memory buffer boundaries |
| **Replay** | Replaying captured legitimate requests or tokens |
| **Privilege escalation** | Gaining higher permissions than authorized |
| **CSRF** | Tricking authenticated user into unwanted action |
| **Directory traversal** | Accessing files outside the web root |

**Privilege Escalation Types:**
- **Vertical privilege escalation**: gaining higher privileges (user → admin/root)
- **Horizontal privilege escalation**: accessing another user's resources at the same privilege level

---

## Cryptographic Attacks

| Attack | Description |
|---|---|
| **Downgrade attack** | Force the use of a weaker, older protocol (SSL stripping forces HTTPS → HTTP) |
| **Collision attack** | Find two different inputs that produce the same hash output |
| **Birthday attack** | Probability-based attack exploiting the birthday paradox to find hash collisions faster |

- Birthday attack is relevant because hash collisions are far more likely than intuition suggests
- Downgrade attacks target negotiation protocols (TLS handshake, SSH version negotiation)

---

## Password Attacks

| Attack | Description | Key Characteristic |
|---|---|---|
| **Password spraying** | Try a few common passwords (Password1!, Summer2024!) against many accounts | Avoids account lockout by staying below threshold |
| **Brute force** | Try every possible combination | Time-consuming; blocked by lockout policies |
| **Dictionary attack** | Try words from a list of common passwords | Faster than brute force; requires a wordlist |
| **Rainbow table** | Pre-computed hash-to-password lookup table | Fast; defeated by salting hashes |
| **Credential stuffing** | Use username/password pairs leaked from other breaches | Exploits password reuse across services |

- **Password spraying** is the go-to technique against enterprise environments with lockout policies
- **Credential stuffing** exploits the widespread habit of reusing passwords across sites
- **Salting** renders rainbow tables useless by adding a unique random value to each password before hashing

## Indicators of Compromise (IoCs)

IoCs are observable evidence that a compromise has occurred or is in progress.

| IoC | What It May Indicate |
|---|---|
| **Account lockout** | Brute force or password spraying attack |
| **Concurrent sessions from different locations** | Credential theft; someone else is using stolen credentials |
| **Impossible travel** | Login from New York, then London 30 minutes later — account is compromised |
| **Blocked content (AV/IDS alert)** | Malware attempted execution; may indicate partial infection |
| **Unusual resource consumption** | Cryptomining malware, ransomware encryption, worm propagation |
| **Resource inaccessibility** | Ransomware has encrypted files; denial-of-service condition |
| **Out-of-cycle logging** | Activity at 3:00 AM when no employees should be working |
| **Missing logs** | Attacker deleted logs to cover tracks; also a log tampering indicator |
| **New admin accounts** | Attacker created a backdoor account for persistence |
| **Outbound traffic to unusual IPs** | Command-and-control (C2) communication |
| **DNS queries to unusual domains** | C2 communication via DNS tunneling |
| **Unexpected firewall rule changes** | Attacker modifying network access |

## Exam Tips

- **Ransomware** indicator = files with new/unknown extensions, ransom note, high disk I/O — classic scenario
- **Rootkit** = hardest to detect; normal OS tools cannot see it; requires offline scanning or anti-rootkit tools
- **Logic bomb** = insider threat tool; triggers on specific condition; no indicator until detonation
- **Worm** indicator = high network traffic across many hosts rapidly; no user action required to spread
- **On-path attack** = the new CompTIA term for MITM; intercepts traffic between two parties
- **Password spraying** avoids lockout by trying few passwords against many accounts — different from brute force
- **Credential stuffing** exploits password reuse — not cracking, just replaying known credentials
- **Impossible travel** = two logins from geographically distant locations in an impossibly short time
- **Missing logs** = attacker covered tracks; always suspicious when logs for a time period disappear
- **Vertical vs. horizontal** privilege escalation — vertical gains higher level; horizontal accesses peer's resources
