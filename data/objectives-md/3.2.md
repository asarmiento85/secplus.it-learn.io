# 3.2 — Security Principles for Enterprise Infrastructure

## What you need to know

This objective covers how security controls are applied to enterprise network infrastructure — including where devices are placed, how firewalls are classified, what network appliances do, and how secure communication is implemented. The exam tests your ability to select the right tool for the right scenario and understand the difference between active vs passive, inline vs out-of-band, and fail-open vs fail-closed behaviors.

---

## Infrastructure Considerations

### Device Placement

- Where a security device sits in the network topology determines what traffic it sees and can control
- **DMZ (Demilitarized Zone)**: semi-trusted zone between the internet and the internal network; hosts public-facing services (web servers, email, DNS)
- **Trusted internal**: private corporate network; higher implicit trust
- **Untrusted external**: the internet; zero implicit trust
- **Restricted zone**: highest-sensitivity internal segment (HR data, financial systems, OT/ICS); most tightly controlled

### Security Zones Summary

| Zone | Trust Level | Typical Residents |
|------|------------|-------------------|
| External / Internet | Untrusted | Public users, attackers |
| DMZ | Semi-trusted | Web servers, email relays, public DNS |
| Internal | Trusted | Employee workstations, file servers, printers |
| Restricted | Highly trusted | PCI cardholder data, executive systems, ICS |

### Attack Surface

- The sum of all the different points where an attacker could enter or extract data
- **Minimize by**: disabling unused services, closing unused ports, removing unneeded accounts, patching, reducing exposed APIs
- Every enabled service, open port, and active user account is part of the attack surface

### Failure Modes

| Mode | Behavior on Failure | Priority | Use Case |
|------|-------------------|----------|---------|
| **Fail-open** | Allows traffic through when device fails | Availability | Non-critical paths where downtime is unacceptable |
| **Fail-closed / Fail-safe** | Blocks all traffic when device fails | Security | Security-critical chokepoints; regulated environments |

- Most security devices should **fail-closed** — a failed firewall that passes all traffic is worse than downtime

### Device Attributes: Active vs Passive

| Attribute | Description | Example |
|-----------|-------------|---------|
| **Active (inline)** | Device sits in the traffic path; processes every packet | Firewall, IPS |
| **Passive (out-of-band)** | Receives a copy of traffic; cannot block, only alert | IDS, packet capture |
| **Inline** | Traffic flows through the device | Firewall |
| **Tap / Monitor port (SPAN)** | Switch sends copy of traffic to monitoring device | IDS sensor, packet analyzer |

## Network Appliances

### Jump Server / Bastion Host

- A hardened, monitored server that serves as the only entry point for administrators into a sensitive network segment
- Reduces attack surface: admins connect to jump server first, then to internal systems from there
- All admin sessions are logged and auditable through a single choke point
- Also called a **privileged access workstation (PAW)** in some contexts

### Proxy Server

| Type | Direction | Purpose |
|------|-----------|---------|
| **Forward proxy** | Client-side | Client requests go through proxy to internet; content filtering, anonymization |
| **Reverse proxy** | Server-side | Sits in front of servers; load balancing, SSL termination, WAF integration |

- Proxy can perform **SSL inspection** (SSL/TLS interception): decrypts, inspects, re-encrypts HTTPS traffic
- Can enforce content filtering policies, block malware downloads, and log user web activity

### IDS vs IPS

| Feature | IDS | IPS |
|---------|-----|-----|
| Placement | Passive (out-of-band / tap) | Active (inline) |
| Response | Alert only — cannot block | Actively blocks matching traffic |
| Traffic impact | None | Slight latency (inline processing) |
| Risk if misconfigured | False negatives only | False positives block legitimate traffic |
| Types | NIDS (network), HIDS (host) | NIPS (network), HIPS (host) |

- **NIDS/NIPS**: monitors network traffic at a chokepoint
- **HIDS/HIPS**: monitors activity on a single endpoint (process execution, file changes, syscalls)

### HSM (Hardware Security Module)

- Dedicated physical hardware device for cryptographic operations and secure key storage
- Performs: key generation, key storage, encryption/decryption, digital signing
- Keys never leave the HSM in plaintext — operations are performed inside the device
- Required for PKI roots of trust, payment card HSMs (PCI-DSS), code signing
- More secure than software-based key storage (keys are protected against OS-level compromise)

### Log Aggregators and SIEM

- **Syslog**: standard protocol for sending log messages from devices to a central server (UDP 514, TCP 514/6514)
- **Log aggregators**: collect and normalize logs from many sources
- **SIEM (Security Information and Event Management)**: aggregates, correlates, and alerts on log data in real time
  - Examples: Splunk, IBM QRadar, Microsoft Sentinel
  - Provides centralized visibility, threat detection, and compliance reporting

## Port Security: IEEE 802.1X

- **IEEE 802.1X**: port-based Network Access Control (NAC) — requires authentication before granting network access at the switch port level
- **EAP (Extensible Authentication Protocol)**: the authentication framework used by 802.1X
- **Components**:
  - **Supplicant**: the device requesting access (laptop, phone)
  - **Authenticator**: the switch or wireless AP enforcing access
  - **Authentication server**: typically a RADIUS server (FreeRADIUS, Microsoft NPS)
- Prevents unauthorized devices from plugging in and getting network access even if they're physically connected

## Firewall Types

| Firewall Type | Layer | How It Works | Strengths |
|--------------|-------|-------------|----------|
| **Packet filtering** | L3/L4 | Checks IP/port headers; stateless | Fast, simple; no connection tracking |
| **Stateful** | L3/L4 | Tracks TCP connection state; allows return traffic | More secure than packet filtering |
| **WAF** (Web Application Firewall) | L7 (HTTP) | Understands HTTP; blocks SQLi, XSS, CSRF | Protects web apps specifically |
| **NGFW** (Next-Gen Firewall) | L3–L7 | DPI, app awareness, IPS, SSL inspection, user identity | Most comprehensive; highest overhead |

### Stateless vs Stateful

- **Stateless (packet filtering)**: examines each packet independently against ACL rules; does not track connection state
- **Stateful**: maintains a connection state table; knows which return packets are legitimate responses to allowed outbound connections

### WAF (Web Application Firewall)

- Sits in front of web applications (Layer 7)
- Understands HTTP/HTTPS requests and responses
- Blocks: SQL injection, cross-site scripting (XSS), CSRF, path traversal, command injection
- Can operate in **positive security model** (whitelist allowed requests) or **negative security model** (blacklist known bad patterns)

### NGFW (Next-Generation Firewall)

- Combines traditional stateful firewall with:
  - **Deep Packet Inspection (DPI)**: reads payload content, not just headers
  - **Application awareness**: identifies applications regardless of port (e.g., recognizes Zoom on any port)
  - **Integrated IPS**: blocks known attack patterns inline
  - **SSL/TLS inspection**: decrypts and inspects encrypted traffic
  - **User identity awareness**: policies based on user identity, not just IP address
- Examples: Palo Alto Networks, Fortinet FortiGate, Cisco Firepower, Check Point

### Layer 4 vs Layer 7 Firewalls

| Layer | What It Uses for Decisions | Capabilities |
|-------|--------------------------|-------------|
| **Layer 4** | Source/dest IP, ports, TCP state | Fast, cannot inspect content |
| **Layer 7** | Application protocol content | Slow but rich inspection; blocks app-layer attacks |

## Secure Communication and Access

### VPN (Virtual Private Network)

- Encrypts traffic over a public (untrusted) network to create a private communications channel
- **Site-to-site VPN**: connects two networks (e.g., headquarters to branch office); always-on
- **Remote access VPN**: connects individual users to the corporate network; on-demand
- **Protocols**: IPsec (standard, L3), SSL/TLS VPN (uses HTTPS port 443, easier through firewalls), WireGuard
- **Split tunneling**: only corporate-bound traffic goes through VPN; internet traffic goes direct — tradeoff of visibility vs performance

### TLS (Transport Layer Security)

- Cryptographic protocol for encrypting data in transit at the transport layer
- Replaced SSL (SSL 3.0 deprecated due to POODLE attack)
- **TLS 1.2**: widely supported; requires cipher suite negotiation
- **TLS 1.3**: faster (1-RTT handshake), removed weak cipher suites, forward secrecy required
- Used by HTTPS, SMTPS, IMAPS, FTPS, and most modern protocols

### SD-WAN (Software-Defined WAN)

- Software-defined approach to managing wide-area network connections
- Dynamically selects the best path (MPLS, broadband, LTE) based on application requirements
- Built-in security features: encryption of WAN traffic, policy-based routing, centralized management
- Reduces dependency on expensive MPLS circuits

### SASE (Secure Access Service Edge)

- Cloud-delivered architecture combining network security and WAN into a single service
- Components: SD-WAN + CASB + SWG (Secure Web Gateway) + ZTNA + FWaaS
- Security policies are enforced at the cloud edge, close to the user — regardless of location
- Aligns with zero trust principles: no implicit trust for users or devices

## Defense in Depth and Control Selection

- **Defense in depth**: layer multiple security controls so that no single control failure causes a breach
  - Network layer: firewall + IPS
  - Host layer: EDR + HIPS
  - Data layer: encryption + DLP
  - Identity layer: MFA + PAM
- **Least privilege**: grant only the access required for the specific role or function
- **Separation of duties**: no single person should have complete control over a sensitive process (requires two people)
- **Zero trust**: never trust, always verify — every access request is authenticated and authorized regardless of network location

## Exam Tips

- Know the difference between IDS (passive, alerts only) and IPS (inline, blocks actively) — this is a frequent scenario question
- Fail-closed = security priority (blocks on failure); fail-open = availability priority (allows on failure)
- Jump server / bastion host = the single hardened admin access point into a sensitive network segment
- 802.1X uses EAP and requires a RADIUS server as the authentication backend
- HSM = hardware key storage; keys never leave the HSM in plaintext
- WAF operates at Layer 7 and specifically targets web application attacks (SQLi, XSS)
- NGFW adds DPI, application awareness, IPS, SSL inspection, and user identity on top of stateful firewall
- SASE = cloud-delivered security + SD-WAN; zero trust enforcement at the cloud edge
- Active device = inline, can block; passive device = tap/SPAN port, monitors a copy, cannot block
- Stateless firewall checks headers only; stateful firewall tracks connection state and allows legitimate return traffic
