# 2.5 — Mitigation Techniques Used to Secure the Enterprise

## What you need to know

Mitigation techniques are security controls and processes applied to reduce the likelihood or impact of a threat exploiting a vulnerability. The Security+ exam tests your ability to match the right mitigation to a given threat scenario. You need to understand both broad enterprise strategies (segmentation, access control, monitoring) and specific technical hardening techniques applied at the endpoint and infrastructure level.

---

## Segmentation

Segmentation divides a network or environment into separate zones to limit the blast radius of a compromise.

| Technique | Description |
|---|---|
| **VLAN (Virtual LAN)** | Logically separates network segments at Layer 2; keeps traffic isolated even on same physical switches |
| **Subnetting** | Divides IP address space into smaller networks; enables firewall rules between zones |
| **DMZ (Demilitarized Zone)** | Network segment that hosts internet-facing services (web, email) isolated from the internal network |
| **Microsegmentation** | Fine-grained segmentation applied to individual workloads, often in software-defined networking and cloud |

**Why it matters:**
- Limits **lateral movement** — if an attacker compromises one segment, they cannot freely reach others
- Contains ransomware spread, worm propagation, and insider threat damage
- DMZ prevents direct internet access to internal systems

---

## Access Control

Access control defines who or what can access resources and what actions they can take.

| Control Type | Level | Description |
|---|---|---|
| **ACLs (Access Control Lists)** | Network / file system | Rules specifying which IP addresses, users, or services can access a resource |
| **Permissions** | File / folder / application | Define read, write, execute rights for users and groups |
| **Role-Based Access Control (RBAC)** | Application / OS | Permissions assigned to roles; users assigned to roles |
| **Mandatory Access Control (MAC)** | OS / government | System-enforced labels; users cannot override |
| **Discretionary Access Control (DAC)** | File system | Owner controls permissions |

- **Principle of least privilege**: grant only the minimum access needed for a user or system to perform its function
- Reduces the **blast radius** — if account is compromised, attacker can only reach what that account can reach
- Review and revoke access regularly; remove access immediately when an employee leaves

## Application Allow List (Whitelisting)

| Approach | Description |
|---|---|
| **Allow list (whitelist)** | Only explicitly approved applications are permitted to run |
| **Block list (blacklist)** | Known malicious applications are blocked; everything else is allowed |

- Allow listing is **far more secure** but requires more administrative overhead
- Prevents unauthorized software, malware, and shadow IT from executing
- Implementation: AppLocker (Windows), WDAC (Windows Defender Application Control), MDM policies
- Blocks zero-day malware that hasn't been added to a block list yet

---

## Isolation

Complete separation of a system from other systems and networks.

| Technique | Description |
|---|---|
| **Air gap** | No network connection whatsoever; physically isolated system |
| **Sandbox** | Isolated environment for executing untrusted code; behavior monitored; cannot affect host |
| **Network isolation** | VLAN or firewall rules preventing all communication to/from a specific host |

- Air-gapped systems are used for the most sensitive environments (nuclear, defense, SCADA)
- Sandboxing is used for malware analysis and executing untrusted email attachments
- Network isolation is used during incident response to contain a compromised host

---

## Patching

Patching is one of the most effective and consistently tested mitigations on the Security+ exam.

| Aspect | Detail |
|---|---|
| **Purpose** | Eliminate known vulnerabilities before attackers can exploit them |
| **Patch Tuesday** | Microsoft's monthly patch release cycle (second Tuesday of each month) |
| **Emergency patches** | Out-of-cycle patches for critical zero-day vulnerabilities |
| **Patch management** | Process for testing, approving, and deploying patches in a controlled manner |
| **Testing** | Patches should be tested in a non-production environment before wide deployment |

- Unpatched systems remain vulnerable to **known, publicly disclosed** CVEs
- Prioritize patches based on CVSS score and exploitability
- Include firmware, drivers, third-party applications — not just the OS
- Track patching compliance with vulnerability scanners (Nessus, Qualys)

## Encryption

| Use Case | Technology |
|---|---|
| **Data in transit** | TLS 1.2/1.3, HTTPS, VPN (IPsec, SSL/TLS), SFTP |
| **Data at rest** | BitLocker (Windows), FileVault (macOS), LUKS (Linux), self-encrypting drives (SED) |
| **Data in use** | Confidential computing, hardware enclaves (Intel SGX) |
| **Email** | S/MIME, PGP |
| **Full disk encryption** | Encrypts entire drive; protects data if device is stolen |

- Encryption protects **confidentiality** — even if data is stolen, it is unreadable without the key
- Key management is as important as the encryption itself — protect the keys

## Monitoring

Continuous monitoring provides visibility into what is happening in the environment.

| Tool | Purpose |
|---|---|
| **SIEM** (Security Information and Event Management) | Aggregates logs, correlates events, alerts on anomalies |
| **IDS** (Intrusion Detection System) | Detects malicious traffic/activity; passive (alerts only) |
| **IPS** (Intrusion Prevention System) | Detects AND blocks malicious traffic; inline |
| **EDR** (Endpoint Detection and Response) | Agent on endpoints; behavioral monitoring, threat hunting, response |
| **Log aggregation** | Centralized collection of logs from all systems for analysis and retention |
| **NetFlow analysis** | Analyzes network traffic patterns for anomalies |

- Without monitoring, attackers can operate undetected for months (as APTs do)
- SIEM enables correlation of events across multiple systems to identify attack patterns
- Monitoring supports compliance requirements (log retention) and incident investigation

## Least Privilege

- Grant users, applications, and systems **only the permissions they need** to perform their function
- Review permissions regularly; remove permissions that are no longer needed
- Apply to:
  - User accounts (no admin rights for regular users)
  - Service accounts (run with minimum required OS permissions)
  - Applications (request only needed permissions)
  - Network devices (limit management access)
- **Just-in-time (JIT) access**: grant elevated privileges only when needed and automatically revoke them afterward
- Reduces the blast radius of a compromised account

## Configuration Enforcement

Ensuring systems are configured securely and consistently across the enterprise.

| Tool / Framework | Description |
|---|---|
| **CIS Benchmarks** | Community-developed hardening guides for OS, cloud, applications |
| **STIG** (Security Technical Implementation Guide) | DoD hardening standards; very stringent |
| **Group Policy (GPO)** | Windows-native mechanism for enforcing configuration across a domain |
| **MDM** (Mobile Device Management) | Enforces security policies on mobile devices (encryption, PIN, app restrictions) |
| **Configuration management tools** | Ansible, Puppet, Chef — automate and enforce desired state configuration |

- Configuration drift is the gradual deviation from a secure baseline — enforcement prevents this
- Regular audits compare current configuration to approved baseline

---

## Decommissioning

Proper disposal of hardware and software to prevent data leakage.

| Step | Description |
|---|---|
| **Data sanitization** | Securely erase data before disposal (DoD 5220.22-M standard, NIST 800-88) |
| **Degaussing** | Strong magnetic field destroys data on magnetic media |
| **Physical destruction** | Shredding, crushing, or incinerating storage media |
| **Certificate of destruction** | Document from third-party disposal company confirming destruction |
| **License reclamation** | Return or transfer software licenses |

- Hard drives and SSDs must be sanitized before disposal or reuse
- Simply deleting files or formatting does not securely erase data
- NIST 800-88: clear (overwrite), purge (degauss/cryptographic erase), destroy (physical)

## Hardening Techniques

Hardening reduces the attack surface of a system by eliminating or securing unnecessary components.

| Technique | Description |
|---|---|
| **Encrypt storage** | Full disk encryption (BitLocker, FileVault) protects data if device is stolen |
| **Endpoint protection** | Antivirus, EDR, NGAV — detects and responds to malware |
| **Host-based firewall** | Windows Defender Firewall, iptables — filters traffic at the individual host level |
| **HIDS** (Host-based IDS) | Monitors system files, logs, and processes for signs of compromise |
| **Disable unused ports/services** | Close services not needed; reduces attack surface |
| **Change default credentials** | Immediately change factory default usernames and passwords |
| **Remove unnecessary software** | Uninstall applications not needed for the system's function |
| **OS hardening** | Apply CIS benchmarks or STIGs; disable legacy protocols (SMBv1, Telnet) |
| **Secure boot** | UEFI Secure Boot verifies bootloader integrity; prevents bootkits |
| **MFA enforcement** | Require a second factor for all administrative access |

## Mitigation Mapping to Threats

| Threat | Primary Mitigations |
|---|---|
| Ransomware | Backup and recovery, network segmentation, email filtering, EDR, least privilege |
| Phishing | Email filtering, user awareness training, MFA, DMARC/DKIM/SPF |
| Insider threat | Least privilege, DLP, UBA, separation of duties, access logging |
| SQL injection | Parameterized queries, WAF, input validation |
| Malware | Endpoint protection, application allow list, patching |
| DDoS | Rate limiting, CDN, upstream filtering, scrubbing centers |
| Password attacks | MFA, account lockout, password complexity, credential monitoring |
| Weak encryption | Use strong algorithms (AES-256, TLS 1.3), disable deprecated protocols |
| Misconfiguration | Configuration enforcement (CIS/STIG), automated scanning |

## Exam Tips

- **Segmentation limits lateral movement** — if a question involves ransomware spreading across the network, segmentation is a key mitigation
- **Patching is the #1 mitigation for known vulnerabilities** — unpatched = vulnerable; patch management includes firmware and third-party apps
- **Application allow listing** is stronger than block listing — only approved apps run; blocks unknown malware
- **Air gap = no network** — used for the most sensitive systems; can't be remotely attacked
- **Least privilege** reduces blast radius — if a question asks how to limit damage from a compromised account, least privilege is the answer
- **Encryption at rest** = BitLocker/FileVault; **in transit** = TLS/VPN — know which technology applies to which scenario
- **Decommissioning** = data sanitization before disposal; NIST 800-88 defines the methods (clear, purge, destroy)
- **SIEM** aggregates and correlates logs; **EDR** protects and responds at the endpoint — both are monitoring tools but serve different layers
- **Host-based firewall** filters traffic at the individual host; **network firewall** filters at the perimeter — exam may test which applies
- **Configuration enforcement** (Group Policy, MDM, Ansible) prevents configuration drift and ensures consistent security baselines
