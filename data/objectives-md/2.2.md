# 2.2 — Threat Vectors and Attack Surfaces

## What you need to know

A **threat vector** is the method or pathway an attacker uses to gain unauthorized access to a system or network. An **attack surface** is the total sum of all possible entry points an attacker can exploit. The Security+ exam tests your ability to recognize threat vectors by their characteristics and know the appropriate defenses. Human vectors (social engineering) receive heavy emphasis because people are consistently the weakest link in security.

---

## Message-Based Vectors

| Vector | Description | Key Defense |
|---|---|---|
| **Email (Phishing)** | Mass unsolicited emails with malicious links or attachments; fake login pages | Email filtering, user awareness training, SPF/DKIM/DMARC |
| **SMS (Smishing)** | Phishing via text message; may link to credential-harvesting site | User awareness; don't click unsolicited links |
| **IM / Chat** | Malicious links via Slack, Teams, WhatsApp, Discord | Application controls, link scanning |
| **Business Email Compromise (BEC)** | Spoofed or compromised executive email authorizing wire transfers or data sharing | Multi-person authorization, out-of-band verification |

- BEC is a **high-value** attack — losses are often in the millions; the attacker impersonates a CEO or CFO
- Phishing variants: **spear phishing** (targeted), **whaling** (executives), **vishing** (voice), **smishing** (SMS)

---

## Image-Based Vectors

- **Steganography**: hiding malicious data or code inside an image file
- **Malformed images**: crafted to exploit vulnerabilities in image-rendering libraries (e.g., buffer overflow in image parser)
- Difficult to detect because image files appear normal and may pass content filters
- Defense: keep image-rendering software patched; use content inspection tools

---

## File-Based Vectors

| Vector | Description |
|---|---|
| **Malicious Office macros** | Word/Excel files with embedded VBA macros that download and execute malware |
| **Malicious PDFs** | PDFs with embedded JavaScript or exploits targeting PDF readers |
| **Infected downloads** | Legitimate-looking software bundled with malware (Trojans) |
| **Archive files** | ZIP/RAR files used to bypass AV scanning by encrypting malicious payload |

- Macros are disabled by default in modern Office; attackers use social engineering to convince users to enable them
- Defense: disable macros, sandbox file opening, endpoint protection

---

## Voice Call Vectors

- **Vishing** (Voice Phishing): attacker calls victim pretending to be IT support, IRS, bank, etc.
- **Pretexting**: fabricated backstory used to justify the request (e.g., "I'm from IT and need your password to fix your account")
- Difficult to defend against technically — user awareness is the primary control
- **Caller ID spoofing** makes calls appear to come from legitimate numbers

---

## Removable Device Vectors

| Vector | Description |
|---|---|
| **USB drop attack** | Attacker leaves infected USB drives in parking lots; curiosity causes victims to plug them in |
| **O.MG cable** | Looks like a normal charging cable but contains a hidden implant for remote access |
| **Infected removable media** | USB drives, SD cards, CDs with malware pre-loaded |
| **HID spoofing** | USB device emulates a keyboard and types malicious commands when inserted |

- Defense: disable USB ports, use endpoint policies to block removable media, user awareness

---

## Vulnerable Software Vectors

| Type | Description |
|---|---|
| **Client-based** | Installed application with a known vulnerability (e.g., outdated browser, Acrobat Reader) |
| **Agentless** | Web-based application accessed via browser; no local install required to attack |

- Unpatched software = persistent vulnerability; patch management is the primary defense
- **End-of-life (EOL) / unsupported systems**: vendor no longer releases patches; any discovered vulnerability is permanent
- Defense: aggressive patch management, replace EOL systems, application allow-listing

---

## Unsecure Network Vectors

| Vector | Description |
|---|---|
| **Open WiFi** | Public hotspots with no encryption; traffic easily intercepted |
| **Rogue Access Point** | Attacker-controlled AP mimicking a legitimate network; user connects to attacker |
| **Evil Twin** | Specific type of rogue AP with same SSID as legitimate network |
| **Bluetooth — Bluejacking** | Sending unsolicited messages via Bluetooth |
| **Bluetooth — Bluesnarfing** | Unauthorized access to data on a Bluetooth device |
| **Bluetooth — Bluebugging** | Taking full control of a Bluetooth device |

- Defense: VPN on public WiFi, WPA3 for enterprise WiFi, disable Bluetooth when not needed

---

## Open Service Ports

- Every open port is a potential entry point — **attack surface reduction** means closing unnecessary ports
- **Principle of least functionality**: systems should only run the services they need
- Common risky open ports: Telnet (23), FTP (21), RDP (3389), SMB (445)
- Port scan with Nmap reveals open ports; attackers do this during reconnaissance
- Defense: host-based firewall, network firewall, periodic port scans to audit exposure

---

## Default Credentials

- Devices and software ship with factory-default usernames and passwords (admin/admin, admin/password)
- **Default credential lists are publicly available** — attackers automate login attempts against known defaults
- Common targets: routers, switches, IP cameras, printers, IoT devices, web application admin panels
- Defense: **change all default credentials immediately** upon deployment

---

## Supply Chain Vectors

| Vector | Description | Example |
|---|---|---|
| **MSP compromise** | Attack managed service provider to reach all their clients | SolarWinds SUNBURST (2020) |
| **Vendor compromise** | Compromise software or hardware through the vendor's build/distribution pipeline | |
| **Hardware tampering** | Physical implants in hardware during manufacturing or shipping | JTAG implants, firmware trojans |
| **Software dependencies** | Malicious or vulnerable third-party libraries included in applications | Log4Shell (Log4j), XZ Utils backdoor |

- Supply chain attacks are particularly dangerous because the trusted update mechanism becomes the delivery vehicle
- Defense: vendor vetting, software bill of materials (SBOM), code signing verification

---

## Human Vectors (Social Engineering)

Social engineering exploits human psychology rather than technical vulnerabilities.

| Technique | Definition |
|---|---|
| **Phishing** | Mass email with malicious links or fake login pages |
| **Spear phishing** | Targeted phishing at a specific named individual |
| **Whaling** | Spear phishing targeted at executives (CEO, CFO) |
| **Vishing** | Voice phishing via phone call |
| **Smishing** | SMS/text message phishing |
| **Impersonation** | Pretending to be a trusted person (IT support, vendor, co-worker) |
| **Business Email Compromise (BEC)** | Spoofed or compromised exec email to authorize fraud |
| **Pretexting** | Fabricated scenario to justify a data request |
| **Watering hole** | Infecting a website the target is likely to visit |
| **Brand impersonation** | Fake website or email that mimics a known brand |
| **Typosquatting** | Registering a domain similar to a legitimate one (paypa1.com, g00gle.com) |
| **Misinformation** | Spreading false information (unintentional or deliberate) |
| **Disinformation** | Deliberately spreading false information for manipulation |

### Social Engineering Principles (Why It Works)

- **Authority**: claiming to be someone with power (CEO, IT director, IRS)
- **Urgency / Scarcity**: "Act now or your account will be suspended"
- **Intimidation**: threatening consequences if the victim doesn't comply
- **Social proof**: "Everyone else has already done this"
- **Familiarity / Liking**: building rapport before making a request
- **Trust**: exploiting established relationships

---

## Attack Surface Summary

| Attack Surface Area | Key Risks |
|---|---|
| Email / messaging | Phishing, malware delivery, BEC |
| Web browsers | Drive-by downloads, malicious links, XSS |
| Remote access (RDP, VPN) | Brute force, credential stuffing, exploitation |
| IoT / embedded devices | Default credentials, no patch support |
| Cloud services | Misconfiguration, shared access keys |
| Endpoints | Unpatched software, USB attacks, malware |
| People | Social engineering, insider threats |

---

## Exam Tips

- **Spear phishing** = targeted at specific person; **whaling** = targeted at executives — know both
- **Vishing** = voice, **smishing** = SMS; "v" for voice, "sm" for SMS
- **Typosquatting** registers domains with common typos — exam often gives a fake URL as a clue
- **Watering hole** attacks compromise sites the *victim* visits — attacker comes to the victim indirectly
- **BEC** is a financial fraud attack — always involves fake authorization for money transfers or data sharing
- **Supply chain** = SolarWinds; the trusted software update was the vector — know this attack pattern
- **O.MG cable** is the USB attack that looks like a normal cable — used in physical attacks
- **End-of-life systems** = permanent vulnerability; no patch will ever exist — must replace or isolate
- **Default credentials** are publicly listed; change them immediately — no exceptions
- Social engineering succeeds by exploiting human psychology, not technical weaknesses
