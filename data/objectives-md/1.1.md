# 1.1 — Security Controls

## What you need to know

Security controls are safeguards or countermeasures used to protect the confidentiality, integrity, and availability of assets. The Security+ exam tests controls along **two independent axes**: the **category** (who implements it / what domain it lives in) and the **type** (what the control does functionally). A single control — like a security camera — can have both a category (Physical) and a type (Detective). Being able to assign both dimensions simultaneously is an exam favorite.

---

## Control Categories

Categories describe **where** a control lives or **who** is responsible for implementing it.

| Category | Definition | Examples |
|---|---|---|
| **Technical** | Implemented by technology / hardware / software | Firewalls, IDS/IPS, encryption, MFA, antivirus, access control lists |
| **Managerial** | Implemented through policies, procedures, and oversight | Security policies, risk assessments, security awareness training programs, audits |
| **Operational** | Implemented by people and day-to-day processes | Security guards, background checks, change management procedures, incident response procedures |
| **Physical** | Tangible, real-world controls that protect physical access | Locks, fences, bollards, cameras, badge readers, mantraps, lighting |

### Technical Controls — Detail
- **Firewalls** — filter traffic at network perimeter
- **Encryption** — protects data confidentiality at rest and in transit
- **Multi-Factor Authentication (MFA)** — requires two or more verification factors
- **IDS/IPS** — monitors and optionally blocks malicious traffic
- **Access Control Lists (ACLs)** — determine who/what can access resources

### Managerial Controls — Detail
- **Security Policies** — written rules governing acceptable use, data handling, etc.
- **Risk Assessments** — identify, analyze, and prioritize risks
- **Security Awareness Training** — educates users about threats and proper behaviors
- **Audits and Reviews** — verify that controls are functioning as intended
- **Background Checks** — vetting of personnel before hire

### Operational Controls — Detail
- **Security Guards** — human patrols and monitoring
- **Procedures and Standard Operating Procedures (SOPs)** — documented step-by-step instructions
- **Separation of Duties** — no single person has end-to-end control of a critical process
- **Incident Response Plans** — documented process for handling security incidents

### Physical Controls — Detail
- **Locks** — doors, server racks, cable locks
- **Fencing and Bollards** — perimeter protection
- **Cameras (CCTV)** — video monitoring of physical spaces
- **Badge Readers** — electronic access control for doors
- **Lighting** — deters criminal activity in dark areas

## Control Types

Types describe **what** a control does functionally — its purpose or behavior.

| Type | Definition | Goal | Examples |
|---|---|---|---|
| **Preventive** | Stops an attack or incident from occurring | Block the threat before it happens | Firewalls, encryption, MFA, locks, fences |
| **Deterrent** | Discourages attackers from attempting | Make the target appear not worth attacking | Warning signs, security guards (presence), lighting, guard dogs |
| **Detective** | Identifies and records that an attack occurred | Discover threats during or after the fact | IDS, security cameras, audit logs, SIEM |
| **Corrective** | Reduces the impact after an incident; restores normal operations | Fix and recover | Patches, backups restore, incident response, system reimaging |
| **Compensating** | Alternative control used when the primary cannot be implemented | Fill a security gap when ideal control isn't feasible | Increased monitoring when MFA can't be deployed; network segmentation for unpatched legacy systems |
| **Directive** | Instructs or mandates specific behavior | Guide behavior through policy or law | Security policies, regulations, acceptable use agreements, laws |

### Key Distinctions Between Types

**Preventive vs. Deterrent:**
- Preventive *actually stops* the attack (firewall blocks packet)
- Deterrent *discourages* the attack but doesn't physically stop it (sign saying "area under surveillance")

**Detective vs. Preventive:**
- Detective discovers after (or during) the fact — IDS alerts you that an intrusion happened
- Preventive stops it before it happens — IPS blocks the intrusion attempt

**Corrective vs. Compensating:**
- Corrective *fixes* something after an incident (restore from backup)
- Compensating *substitutes* for a missing primary control (extra logging because you can't patch a legacy system)

**Directive:**
- Purely behavioral — tells people what they must do
- Examples: "You must use a strong password" policy; HIPAA; GDPR; Acceptable Use Policy

## Combining Categories and Types

Every control has **both** a category AND a type. This is where exam questions get tricky.

| Control | Category | Type(s) |
|---|---|---|
| Security camera (CCTV) | Physical | Detective (records), Deterrent (visible presence) |
| Security guard | Physical / Operational | Deterrent, Detective, Preventive |
| Firewall | Technical | Preventive |
| IDS (Intrusion Detection System) | Technical | Detective |
| IPS (Intrusion Prevention System) | Technical | Preventive |
| Encryption | Technical | Preventive |
| MFA | Technical | Preventive |
| Security awareness training | Managerial / Operational | Preventive (reduces human error) |
| Security policy | Managerial | Directive |
| Audit logs | Technical | Detective |
| Acceptable Use Policy (AUP) | Managerial | Directive |
| Fence | Physical | Preventive, Deterrent |
| Bollard | Physical | Preventive |
| Warning sign | Physical | Deterrent |
| Backup restoration | Operational / Technical | Corrective |
| Incident response plan | Managerial / Operational | Corrective |
| Increased monitoring (legacy system) | Technical | Compensating |
| Network segmentation (legacy) | Technical | Compensating |
| Lock | Physical | Preventive |
| Badge reader | Physical / Technical | Preventive |
| Lighting | Physical | Deterrent, Preventive |

## Scenario Practice

**Q: A hospital cannot patch a critical legacy medical device. What type of control would they implement to address the risk?**
A: Compensating — they use an alternative (e.g., network segmentation, increased monitoring) because the primary control (patching) isn't feasible.

**Q: A security guard patrols the parking lot. What category AND type is this?**
A: Category = Physical (and Operational — it's a person doing a procedure); Type = Deterrent (presence discourages), Detective (observes activity).

**Q: An organization requires all employees to sign an Acceptable Use Policy. What type of control is this?**
A: Directive — it instructs/mandates behavior.

**Q: After a ransomware attack, the IT team restores data from clean backups. What type of control is the backup restore?**
A: Corrective — it fixes the damage after the incident.

## Exam Tips

- The exam almost always asks for **both** category and type — always think in both dimensions.
- A **camera** is Physical (category) and Detective (type) — but if it's visible, it's also Deterrent.
- **Deterrent** vs **Preventive** is a classic trap: a guard *deters* but doesn't always *prevent*; a fence *prevents* physical access.
- **Compensating controls** exist when the primary control is not feasible — look for language like "cannot be implemented," "legacy system," "workaround."
- **Directive controls** are purely behavioral — policies, laws, regulations, and training mandates fall here.
- **Corrective** = fixes after the fact. Think: restore from backup, apply patch after breach, reimage a system.
- **Detective** = identifies that something happened. IDS, SIEM, audit logs, security cameras all detect.
- **IDS** = Detective; **IPS** = Preventive. Know this distinction cold.
- **Managerial** controls live in documents and meetings — risk assessments, policies, reviews.
- **Operational** controls are implemented by people following procedures — guards, SOPs, awareness programs.
- Watch for "which of the following is an example of a compensating control" — the answer will involve an organization that cannot use the best-practice solution.
